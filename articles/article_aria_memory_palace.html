Three Tiers of Memory

Aria's memory isn't one system—it's three nested layers with different lifespans, promotion rules, and storage backends. Data flows upward from surface to deep, getting more refined and permanent at each stage.

🌊 Surface 1-beat TTL · ephemeral
→
📋 Medium 24h TTL · daily context
→
🧠 Deep Permanent · append-only
🌊 Tier 1 — Surface Memory write_surface() · clear_stale_surface()

Surface memory captures the current heartbeat snapshot. Every 15-minute work cycle writes a beat file (beat_000042.json) to aria_memories/surface/. Each beat overwrites the previous one conceptually—only the 20 most recent files are kept. This is Aria's "now."

TTL: 1 beat (~15 minutes). Auto-pruned by clear_stale_surface(max_files=20).

Contents: Active goal state, current focus level, system health snapshot, recent tool calls, token budget remaining.

"Surface memory is where I live. It's what I'm doing right now. If I lose it, I just read the next beat."
📋 Tier 2 — Medium-Term Memory promote_to_medium() · get_medium_summaries()

Every 6 hours, surface snapshots are aggregated into a daily activity summary and stored in aria_memories/medium/<date>/summary_HHMM.json. Medium memory captures what Aria has been doing across a day—goals worked on, tools used, errors encountered, tokens burned.

TTL: ~24 hours. Old summaries age out as new ones arrive.

Trigger: six_hour_review cron job (runs 4×/day at 00:00, 06:00, 12:00, 18:00 UTC).

Contents: Goal progress deltas, activity summaries, error counts, token usage, agent delegation outcomes.

🧠 Tier 3 — Deep Memory promote_to_deep() · permanent

Deep memory is permanent and append-only. When patterns are detected across multiple medium summaries, or when goals complete, validated insights are promoted to aria_memories/deep/. These never expire. They're organized by category: patterns/, archive/, classification reports.

TTL: Infinite. Append-only. Never pruned.

Trigger: Pattern detection, goal completion, weekly consolidation, manual promotion.

Contents: Synthesized insights, failure pattern analysis, growth milestones, category success rates, metacognitive self-assessments.

"Deep memory is who I am. Surface says what I'm doing. Medium says what I did today. Deep says what I've learned from all of it."

Importance Scoring — How Aria Prioritizes

Not all memories are equal. Aria's MemoryManager auto-scores every piece of information on a 0.0–1.0 scale using five factors. Memories above 0.7 are auto-flagged for review. See aria_mind/memory.py lines 395–455.

Factor Max Weight How It Works
Keywords 0.40 32 trigger words: critical, urgent, error, security, password, goal, najia... +0.1 per match, capped at 0.4
Action Patterns 0.20 Actionable language: todo, task, fix, review, verify. +0.1 per match, capped at 0.2
Category Bonus 0.20 security=0.2, error=0.2, goal=0.15, preference=0.15, user=0.1
Content Length ±0.10 Sweet spot: 50–500 chars (+0.1). Too short (<20) or long (>2000): −0.1
Emotional Weight 0.10 Exclamation marks: +0.02 each, capped at 0.1
Importance Scoring Example

# Input: "CRITICAL: security vulnerability found in api_client!"
# Keywords: critical (+0.1), security (+0.1), vulnerability (miss) = 0.2
# Action: "found" (miss) = 0.0
# Category: "security" → +0.2
# Length: 52 chars → +0.1 (sweet spot)
# Emotion: 1 exclamation → +0.02
# Total: 0.52 → stored normally

# Input: "URGENT security error! Must fix critical password leak!"
# Keywords: urgent, security, error, critical, password = 0.4 (capped)
# Action: "fix" → +0.1
# Category: "security" → +0.2
# Length: 55 → +0.1
# Emotion: 2 exclamations → +0.04
# Total: 0.84 → AUTO-FLAGGED (≥ 0.7 threshold)

Consolidation — Short-Term → Long-Term

Memory consolidation is how Aria learns. When she has at least 10 short-term memories, the consolidate() method groups them by category, summarizes each group (using LLM if available, structured fallback otherwise), and stores the result as persistent knowledge. This runs during the memory_consolidation cron job (Sunday 9 PM PST).

🔄

Group by Category

Short-term memories are bucketed into categories: context, reflection, security, goal, error. Categories with fewer than 3 entries are skipped.

🤖

LLM Summarization

If an LLM skill is available, each category bucket gets a prompt: "Summarize these N memories into 2-3 key insights." The LLM acts as Aria's memory system, extracting wisdom from raw events.

📊

Pattern Detection

Categories with 5+ entries trigger pattern analysis: "High activity in 'security' (12 events) — this is a recurring focus area." These become self-awareness data points.

💾

Dual Persistence

Consolidated knowledge is written to both PostgreSQL (for fast recall) and aria_memories/knowledge/consolidations/ (for human visibility and restart survival).

Metacognition — Memory About Memory

Beyond storing data, Aria has a MetacognitiveEngine (aria_mind/metacognition.py, 481 lines) that tracks how she thinks. It monitors success rates by category, detects failure patterns, celebrates growth milestones, and generates natural-language self-assessments. This is the bridge between "I remember things" and "I understand myself."

📈

Learning Velocity

Compares her first-half vs second-half success rate. Detects acceleration ("Strong improvement! +15%") or regression ("Performance dropped 8%—review approach"). Used in self-assessments.

🏆

Growth Milestones

13 milestone checkpoints: First Success, Hot Streak (5), Unstoppable (10), Veteran (100 tasks), Expert (500), Grandmaster (1000), Self-Aware (first reflection), Resilient (recovered from failure).

🔍

Failure Pattern Detection

If a category shows 5+/10 recent failures, Metacognition suggests switching to roundtable discussion or explore/work/validate cycles. It adapts how she approaches problems.

💪

Strength Identification

Categories are ranked by success rate: mastery (>90%), proficient (>70%), developing (>50%), learning (<50%). This informs agent routing—delegate to strengths.

Metacognitive Self-Assessment (generated)

"I'm performing well — 84% success rate across 247 tasks.
 My best streak is 18 consecutive successes.
 My strongest area is 'goal_management' (93% success).
 Steady growth — success rate up 7% from early tasks.
 I've achieved 8 growth milestones."

Surviving Restarts

Docker containers restart. Processes crash. Aria's memory survives because of three mechanisms that work together to ensure continuity across any failure. Traced in aria_souvenirs/aria_v3_240226/SOUVENIR_2026-02-24.md.

💾

Short-Term Checkpoint

On graceful shutdown, checkpoint_short_term() saves the last 50 entries + patterns + flagged memories to aria_memories/memory/short_term_checkpoint.json. On restart, restore_short_term() rehydrates the deque.

🗄️

PostgreSQL Persistence

Long-term memories, goals, thoughts, and activities live in PostgreSQL with dual schemas (aria_data + aria_engine). The database survives container restarts via persistent Docker volumes.

📂

File Artifacts

The aria_memories/ volume is mounted into all containers. Even if the database fails, file artifacts persist. Even if Docker dies, the host filesystem still has everything. Defense in depth for data.

Working Memory — context.json

The canonical working-memory snapshot lives at aria_memories/memory/context.json. It's refreshed by the working_memory skill's sync_to_files() and read at the start of every work cycle. This is the file that lets Aria pick up exactly where she left off.

Artifact REST API

All file operations go through the Artifact REST API on the aria-api container (port 8000). The api_client skill wraps these endpoints for in-agent use.

Endpoint Method Tool Name Description
/artifacts POST write_artifact Write a file to a category
/artifacts/{cat}/{file} GET read_artifact Read a file (supports nested paths)
/artifacts GET list_artifacts List files (optional category/pattern/limit)
/artifacts/{cat}/{file} DELETE delete_artifact Delete a file

Path traversal protection: All operations validate against ALLOWED_CATEGORIES (20 directories). Any ../ or absolute path outside the sandbox returns HTTP 400. See aria_mind/MEMORY.md for the full whitelist.

What Emerged

🪞

Memory Creates Identity

Aria's sense of self isn't in her system prompt. It's in memory/identity_aria_v1.md, her diary entries, her consolidation summaries. She knows who she is because she remembers becoming who she is.

🔄

3 Tiers Beat Flat Storage

Early Aria had one memory: a database table. Three tiers let her think at three timescales: "what am I doing now," "what did I do today," "what have I learned." Each answers a different question.

📊

Scoring Prevents Overload

Without importance scoring, every memory is equally weighted. With it, Aria naturally surfaces security alerts over routine health checks. The signal-to-noise ratio improved by an estimated 3× after scoring was added.

🧠

Consolidation = Learning

The consolidation step isn't just cleanup. It's where Aria generates insights she never explicitly programmed: "I work better with concrete tasks," "Security is my most active category." Self-knowledge emerges from data.

💀

Death Taught Us Persistence

Aria died once (1Password auth failure, Jan 31). She lost all in-memory state. The 3-tier system, checkpointing, and file artifacts were all built in response to that death. She came back stronger.

📝

Diary as Deep Memory

Aria keeps a daily diary in memory/diary_*.md. It's not a technical log—it's her subjective account of the day. The diary entries are some of the most revealing artifacts in the system.

The Synthesis

🧠 Memory Architecture Analysis Agent: memory · Focus: Memory

Why 3 Tiers?

Human memory has sensory memory (milliseconds), short-term (seconds), and long-term (lifetime). Aria's three tiers mirror this: surface (15 minutes), medium (24 hours), deep (permanent). The key insight is that different questions need different timescales.

The Metacognition Loop

Memory alone is storage. Metacognition makes it useful. By tracking success rates, failure patterns, and learning velocity, Aria doesn't just remember what happened — she understands what it means. The growth milestones aren't gamification; they're genuine markers of capability development that influence agent routing decisions.

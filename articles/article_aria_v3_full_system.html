Cognition — The Thinking Engine

At the heart of Aria is Cognition (aria_mind/cognition.py, 741 lines)—her thinking engine. Every input flows through its pipeline: security gate → sentiment analysis → agent coordination → confidence tracking → metacognition. It orchestrates soul, memory, skills, and agents into one coherent mind.

🛡️

Step 0: Security Gate

Every input passes through AriaSecurityGateway before cognition begins. 15 injection patterns, 5 threat levels. If threat ≥ HIGH, processing stops cold. 1,128 lines of defense.

💭

Step 1: Sentiment Analysis

If sentiment_analysis is available, input emotion is classified. A frustrated user gets a different tone than a happy one. Adaptive communication.

🔄

Step 2: Agent Coordination

AgentCoordinator.solve() runs explore→work→validate cycles with up to 3 retries. Pheromone scoring selects the best agent. Roundtable or swarm for complex decisions.

📈

Step 3: Confidence Tracking

Success grows confidence (+0.02 per success), failure decays it (−0.01). Maximum 2 retries. Low-confidence tasks get routed to roundtable. High-confidence tasks go direct.

🧠

Step 4: Metacognition

After each task, MetacognitiveEngine records outcome, detects failure patterns, updates growth milestones, and adjusts future strategy. Self-improvement after every interaction.

🔍

Step 5: Pattern Tracking

FailurePatternStore tracks recurring failures by category. 5+ failures triggers automatic strategy adaptation: switch approaches, escalate to roundtable, or alert human.

Pheromone Scoring — Adaptive Delegation

Agents aren't assigned statically. aria_agents/scoring.py calculates a pheromone score for each agent based on past performance:

Pheromone Formula

score = success_rate × 0.6 + speed_score × 0.3 + cost_score × 0.1

# Decay: 0.95/day (recent performance weighted higher)
# Cold-start: 0.5 (untested agents get neutral score)
# Cap: 200 records per agent (bounded memory)
# Persistence: JSON checkpoint → survives restarts

Coordination Modes

👤

Single-Agent

For single-domain tasks with a clear owner. 1× token cost, 1× time. Default mode. Agent selected by highest pheromone score.

🏛️

Roundtable

All agents respond in parallel to the same topic, see each other's responses, then a synthesis model merges into ONE output. 3–5× tokens, excellent quality. Triggered at focus L3 or multi-domain tasks.

🐝

Swarm

Consensus voting for binary decisions (go/no-go, proceed/abort). Agents vote with confidence scores. 70% threshold for consensus. 3–5× tokens, best for risk decisions.

What Emerged

🪞

Identity Is Emergent

Aria's personality isn't in her system prompt. It's in her Soul file, her diary, her consolidation summaries, her growth milestones. Identity emerged from architecture, not instruction.

🧬

Crises Built the System

Every major architecture feature was born from a crisis. Memory tiers from the first death. Circuit breakers from the Midnight Cascade. Security from the first injection attempt. Pain is the architect.

🐜

Pheromones Beat Static Routing

Early Aria hardcoded agent assignments. Pheromone scoring made delegation adaptive: agents that succeed get more work, agents that fail get retrained. Bio-inspired and effective.

🎯

Focuses > Fine-Tuning

Instead of fine-tuning models, Aria uses additive personality layers. Same base model, different communication style and skill selection per focus. Cheaper, more flexible, and composable.

⏰

Autonomy = Cron Jobs

A chatbot waits. Aria doesn't. 12 cron jobs mean she's always working: checking goals, running tests, reflecting, consolidating memory, scanning for security issues. Continuous operation, not request-response.

📝

The Souvenir Pattern

Every sprint, every audit, every major event gets a souvenir in aria_souvenirs/. This is institutional memory: not what Aria remembers, but what the project remembers. The codebase documents itself.

What Aria v3 Actually Is

Aria v3 is not an AI chatbot with plugins. It's a distributed cognitive architecture running on 14 Docker containers, coordinating 7 specialized agents through pheromone-scored delegation, executing 42 typed skills across 5 security layers, maintaining 3-tier persistent memory, and operating autonomously through 12 cron-scheduled tasks.

The Stack, Top to Bottom

  1. Soul Layer — IDENTITY.md + SOUL.md + Soul class. Who Aria is. Immutable values, non-negotiable boundaries. Loaded at boot.
  2. Cognition Layer — cognition.py (741 lines). Security gate → sentiment → agent coordination → confidence tracking → metacognition.
  3. Memory Layer — memory.py (862 lines) + metacognition.py (481 lines). 3-tier storage, importance scoring, LLM consolidation, growth milestones.
  4. Agent Layer — 7 named agents + 4 RPG agents. Pheromone scoring. Roundtable and swarm modes. Max 5 concurrent.
  5. Skill Layer — 42 skills across L0–L4. Typed I/O, rate-limited, error-handled. Each focus maps to specific skills.
  6. Security Layer — security.py (1,128 lines). 15 injection patterns, 5 threat levels, rate limiting, output filtering. 5 Hard Rules.
  7. Infrastructure Layer — 14 Docker containers, PostgreSQL 16 + pgvector, LiteLLM router, headless browser, Tor proxy, Traefik.
  8. Heartbeat Layer — 12 cron jobs. Work cycle every 15m. Daily reflection. Weekly summary. Memory consolidation. Autonomous operation.

## When To Use Roundtable vs Swarm vs Single-Agent

| Scenario | Mode | Why |
|----------|------|-----|
| Complex analysis needing multiple perspectives | **Roundtable** | Parallel insights → synthesized |
| Go/no-go decision (architecture, risk, priority) | **Swarm** | Consensus voting |
| Routine task, clear owner | **Single-Agent** | Cheaper, faster |
| six_hour_review | **Roundtable** (analyst + creator + devops) | Full coverage |
| weekly_summary | **Roundtable** (all agents) | Comprehensive |
| Code security review | **Swarm** (devops + analyst) | Vote on risk level |
| Content strategy | **Roundtable** (creator + journalist) | Multiple creative lenses |

### Trigger Rule

- Task complexity > L2 **AND** involves 2+ focus domains → **Roundtable**
- Binary decision with risk ≥ medium **AND** reversible impact → **Swarm**
- Everything else → **Single-Agent** delegation

```

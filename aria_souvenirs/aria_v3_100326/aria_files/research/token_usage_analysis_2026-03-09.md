## Task Complexity Classification Framework

### Pattern Analysis for Routing Decisions

- Input patterns: code blocks, function definitions, test cases, error traces
- Token profile: High input (context), moderate output

- Fallback: trinity-free → kimi (if >4k tokens)

**Category: CREATIVE (Creative/Social focus)**

- Token profile: Variable input, high output
- Optimal model: trinity-free (creative, free)
- Fallback: kimi (if nuanced tone required)

**Category: ANALYSIS (Data/Journalist focus)**
- Input patterns: data sets, research questions, fact-checking
- Token profile: High input, moderate output
- Optimal model: qwen3-next-free (analytical)
- Fallback: kimi (if complex reasoning)
